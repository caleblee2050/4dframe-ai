# 4DFrame AI 짝코더 — Vision

> "정해진 답을 따라가는 장난감"이 아닌, **학생이 AI 와 함께 만들어가는 사고의 도구**.

---

## 1. 미션

**4DFrame 부품으로 만든 학생의 작품**에, **AI 와의 협업 창작**을 더해 — 단순한 조립 완성품이 아니라 *생각하고-말하고-수정하는 살아있는 작품*으로 만든다.

핵심 한 줄: **"내가 만든 걸, 내 말로 움직이게 한다."**

---

## 2. 차별화 포인트

### 시중 교구재와의 차이

| 일반 STEM/코딩 장난감 | 4DFrame AI 짝코더 |
|---|---|
| 정해진 명령어 / 카탈로그 | 자연어 — 어떤 표현이든 OK |
| 정해진 노래/효과음 | **AI 즉석 작곡** — 분위기 묘사로 음악 생성 |
| 정해진 동작 시퀀스 | 학생-AI 대화로 동작을 만들어 나감 |
| 결과물 = 완성된 동작 | **결과물 = 사고력 훈련 과정** 자체 |
| 단일 언어 | 한국어 / English / Монгол |

### 핵심 가치: "맞춰 나가는 과정 = 사고력 훈련"

> 완벽한 결과보다 **학생이 AI 의 결과를 보고 "이상해", "더 무섭게"** 하는 반복 자체가 가치.

이 반복 = **메타인지 + 자기 표현 + 추상↔구체 사이 번역** 훈련. 유치원생도 이 과정 안에서 자연스럽게 사고력을 키운다.

---

## 3. 핵심 가치

1. **창작의 즐거움** — AI 가 만든 결과를 학생이 다듬는 권한
2. **부정확함의 미덕** — 음악이 살짝 어색해도 "어떻게 바꿀까?" 가 시작점
3. **단순함 우선** — 유치원생도 손 안 대고 작동 (마이크/손 동작/큰 칩)
4. **다국어 평등** — 한/영/몽 모두 같은 경험. 글로벌 교육 시장 진입 기반
5. **부모-자녀 분리** — 학생은 단순 화면, 어른은 ⚙️ 모달 / 고급 모드로

## 4. 사용자 페르소나

### Primary: 유치원~초등 저학년 학생 (5~9세)
- 자기 작품을 보여주고 자랑하고 싶어 함
- 글자 입력 어려움 → **마이크 + 칩 클릭** 위주
- 결과가 "내가 한 거" 라는 만족감 중요
- 영어/몽골어 환경 학생 포함 (해외 교육 시장)

### Secondary: 부모 / 교사
- 교구재 가이드 노출 최소화 — 학생이 직접 발견하길 원함
- 모터 빡빡하면 ⚙️ 길들이기 가능
- 안전 — 펌웨어 자동 정지, 9V 분리 시 동작 X

### Tertiary: 운영/관리자
- API 비용/호출 통계 (`/admin`)
- 펌웨어 플래시 도구 (`stk500_flash.py`)

## 5. 핵심 사용 시나리오

### 시나리오 A — "**악어가 우리 집을 지켜!**" (5세)
1. 학생이 4DFrame 으로 악어 작품 만듦
2. 쉬운 모드 → 악어 작품 선택
3. 마이크 🎤 → "**우리 집에 도둑이 들어올까봐 무서워. 악어가 지켜줘**"
4. AI: "[whispers]내가 망보고 있을게..." + 거리센서 반응 모드 ON 유도
5. 사람이 가까이 오면 → 악어 입 크게 벌리기 + 죠스 음악
6. 학생: "더 무섭게!" → AI 가 음악 다시 작곡 (저음 sine 점진 가속)

### 시나리오 B — "**오르골 발레리나가 점점 천천히**" (7세)
1. 회전 작품 + 발레리나 인형
2. "오르골 발레리나" 칩 한 번 클릭
3. 음악 + 회전이 정확히 동기화 → 점점 느려지면서 음악도 끝남
4. 학생: "근데 음악이 너무 빨라. 더 천천히" → AI tempo 0.7 로 재실행

### 시나리오 C — "**우리 자동차로 한 바퀴**" (6세)
1. 자동차 작품 → 조이스틱 자동 등장
2. 학생: 조이스틱 살짝 밀기 → 천천히 / 끝까지 → 빠르게
3. 또는 키보드 화살표로 단순 조종
4. "음악 들으며 한 바퀴 돌아!" → AI 가 행진곡 작곡 + drive 시퀀스

### 시나리오 D — "**Hello, my dragon!**" (영어권 7세)
1. 🇺🇸 클릭 → UI/응답/TTS 모두 영어
2. "Make my dragon roar three times" → AI 영어 응답 + 영어 voice + 동작

## 6. 기술 차별화

### 6.1 AI 즉석 작곡 (`tune: 'custom'`)
- AI 가 분위기/리듬/노래 이름 듣고 음표 시퀀스 직접 생성
- 미리 정의된 9개 preset (학교종/반짝반짝/오르골 등) 외에 무한 확장
- 부정확해도 OK — 학생-AI 대화로 다듬어감
- timbre 3종 (square 전자음 / triangle 오르골 / sine 부드러움)

### 6.2 음악-동작 정확 동기화
- `melodyDurationMs` 로 음악 길이 측정 → 동작 V level 진행률 매핑
- 발레리나 오르골: V9→V2 linear 감속, 음악 끝 = 모터 정지
- 학교종/죠스 등 다른 음악 스킬도 같은 패턴

### 6.3 펌웨어 v1.3.5 — 자기 진단 부팅
- BLE baud auto-changer (한 번 부팅 시 BLE 모듈 영구 설정)
- EEPROM 플래그 gate — 한 번만 실행
- 진단 결과 부팅 메시지에 (`BLE:9600->115200`)
- v1.3.3 의 timer1 detach 후 PWM 복원 로직 그대로

### 6.4 다국어 = 1급 시민
- UI / AI 응답 / TTS voice / 스킬 라벨 / say.text 모두 locale 전환
- zustand persist (`4dframe-i18n`) — 새로고침해도 유지

## 9. 핵심 결정 기록

### 왜 customSkills 환경 별로 분리되어 있나?
zustand persist 가 origin 별 localStorage 라 localhost / vercel.app 다름. 같은 환경 안에서는 동기화. D+ 작업으로 서버 sync 가능.

### 왜 BLE 지원 안 하나?
보드 J6 슬롯이 ATmega328 hardware UART (D0/D1) 와 회로상 연결되지 않은 것으로 진단 (펌웨어 v1.3.5 의 baud auto-changer 가 5개 baud 전부 응답 0 검출). USB 와 BLE 가 같은 wire 충돌 외에도, BLE 데이터가 펌웨어까지 도달 안 하는 회로 문제. 보드 v2 에서 SoftwareSerial 또는 별도 라우팅으로 해결 예정.

### 왜 Gemini 가 아닌 Claude 를 자연어 모드에 사용하나?
페르소나 일관성 (학생 친화 톤), JSON 구조화 안정성, system prompt 준수. TTS 만 Gemini.

### 왜 단축키 (C/O/P) 제거?
유치원생은 키보드 단축키 개념 X. 큰 버튼 + 아이콘이 더 직관적.
